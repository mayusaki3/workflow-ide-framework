# 制約・方針

Workflow IDE Framework の開発にあたり、以下の制約・方針を定める。

## 1. 要件分離

機能要件・アーキテクチャ要件・非機能要件を分離して管理する。

## 2. 特定技術への固定依存を避ける

特定 AI・特定 Runtime・特定クラウド専用としない。

## 3. Framework 本体の肥大化を避ける

用途固有機能は Plugin や外部 Runtime として分離可能とする。

## 4. 長期運用を前提とする

保守・テスト・ドキュメント整備・拡張を継続可能な構造とする。

## 5. AI 連携を前提とする

AI 利用を後付けではなく、初期設計段階から考慮する。

## 6. 人間が理解しやすいこと

ノーコード・ローコード利用者でも扱いやすい構造を考慮する。

## 7. 外部連携を前提とする

外部 Runtime・外部プロセス・外部システムとの連携を考慮する。

## 8. HLDocS 準拠

ドキュメント管理は HLDocS に準拠する。
