# アーキテクチャ要件

Workflow IDE Framework のアーキテクチャは、以下の要件を満たすこと。

## 1. Runtime 分離

UI と Runtime は process 分離可能な構成を考慮すること。

## 2. Workflow と Runtime の分離

Workflow 定義と Runtime 実装を分離すること。

## 3. Plugin による分離

用途固有機能は Plugin や外部モジュールとして分離可能であること。

## 4. 疎結合構造

UI・Runtime・Plugin・外部システムは疎結合構造を考慮すること。

## 5. 外部 Runtime 接続

別マシン・別 OS・別 Runtime 環境への接続を考慮すること。

## 6. AI Runtime 切り替え

AI Runtime を切り替え可能な構造を考慮すること。

## 7. GPU 利用構造

GPU Viewport やリアルタイム描画を扱える構造を考慮すること。

## 8. Command 統一

UI・API・外部Runtimeなど複数操作経路を統一可能な構造を考慮すること。

## 9. オフライン利用

ローカル環境のみでも利用可能な構造を考慮すること。

## 10. 開発環境と実行環境の分離

開発者向け環境とエンドユーザー向け実行環境を分離可能であること。
