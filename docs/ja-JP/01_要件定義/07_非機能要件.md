# 非機能要件

Workflow IDE Framework は、以下の非機能要件を満たすこと。

## 1. 拡張性

後から Node・Runtime・Plugin を追加可能であること。

## 2. 再利用性

他プロジェクトでも再利用可能な構造であること。

## 3. 可搬性

複数 OS・複数実行環境で利用可能であること。

## 4. リアルタイム性

ロボット制御・VR・シミュレーションなど、
リアルタイム処理への利用を考慮すること。

## 5. 保守性

長期運用・保守・拡張を継続可能であること。

## 6. 可読性

人間が理解しやすい構造・UI・Workflow 表現を考慮すること。

## 7. オフライン利用

インターネット接続なしでも利用可能な構成を考慮すること。

## 8. 導入容易性

利用者向けに導入支援機能を持つこと。

## 9. 技術非依存

特定 AI・特定 Runtime・特定クラウドへの固定依存を避けること。
