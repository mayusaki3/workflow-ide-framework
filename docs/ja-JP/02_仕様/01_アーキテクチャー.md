# アーキテクチャー

# 1. 概要

Workflow IDE Framework は、Runtime IDE を目的とした Rust + egui ベースの Workflow 実行環境である。  
GPU Viewport・Workflow Runtime・Event / State・Command・Plugin を Core 構造として扱う。

# 2. Core 構造

## 2.1 UI

- egui + eframe を利用する
- Docking UI を利用する
- GPU Viewport を Core UI として扱う
- 複数 UI 部品へ分割可能な構造とする

## 2.2 Runtime

- Runtime は process 分離構造とする
- UI と Runtime を分離する
- 外部 Runtime 接続を考慮する

## 2.3 Workflow

- Node Graph ベースとする
- Workflow 定義は Runtime 非依存とする

## 2.4 Event / State

- Event / State モデルを採用する
- Event Bus は async channel ベースとする
- State Store は Redux 型を採用する

## 2.5 Command

- Command モデルを採用する
- UI・API・WebView・外部Runtime を統一 Command で扱う
- Undo / Redo を考慮する

## 2.6 GPU Viewport

- GPU Viewport は IDE Core 構造として扱う
- Viewport trait により拡張可能とする
- Docking UI と統合する

## 2.7 WebView

- WebView は Support Panel として扱う
- wry を利用する
- Help / Docs / Tutorial / AI Chat 用途を想定する
- IDE 操作は Command 経由で行う

## 2.8 Plugin

- IPC Plugin 構造を採用する
- Plugin は process 分離を考慮する

# 3. 技術構成

| 項目 | 採用 |
|---|---|
| 言語 | Rust |
| GUI | egui |
| Window / Backend | eframe |
| WebView | wry |
| Event Bus | async channel |
| State Store | Redux 型 |
| Workflow | Node Graph |
| Runtime | process 分離 |
| Plugin | IPC Plugin |

# 4. 今後の詳細仕様

詳細仕様は、必要タイミングで以下のように分割する。

- UI 部品
- 実行モデル
- Runtime Lifecycle
- IPC Message
- Viewport Lifecycle
- Plugin Lifecycle
