# WV-08-01_08-05 GTK基本検証

## 目的

GTK統合層の最小構成を段階的に追加し、応答なしが発生する最小条件を特定する。

## WV-08-01 GTK完全無効化

### 目的

GTK関連処理を全て停止し、eframe / egui / egui_dock のみで動作確認を行う。

### 実施内容

* GTK初期化停止
* GTK Window生成停止
* GTKイベント処理停止
* WebView生成停止

### 結果

* Dock移動可能
* Dockサイズ変更可能
* マウス操作可能
* 応答なし発生せず

### 判定

成功

### 知見

GTK未使用状態では問題は発生しない。

---

## WV-08-02 GTK Host Window生成

### 目的

GTK初期化およびGTK Window生成のみで問題が発生するか確認する。

### 実施内容

* gtk::init()
* gtk::Window::new()

### 結果

* GTK Window生成成功
* Dock移動可能
* Dockサイズ変更可能
* マウス操作可能
* 応答なし発生せず

### 判定

成功

### 知見

gtk::init() と gtk::Window::new() は主因ではない。

---

## WV-08-03 Window表示

### 目的

GTK Window表示のみで問題が発生するか確認する。

### 実施内容

* window.show_all()

### 結果

* GTK Window表示成功
* Dock移動可能
* Dockサイズ変更可能
* マウス操作可能
* 応答なし発生せず

### 判定

成功

### 知見

window.show_all() は主因ではない。

---

## WV-08-04 GTK Window保持

### 目的

GTK Window保持のみで問題が発生するか確認する。

### 実施内容

```rust
unsafe {
    GTK_WINDOW = Some(window);
}
```

### 結果

* GTK_WINDOW stored 出力確認
* Dock移動可能
* Dockサイズ変更可能
* マウス操作可能
* 応答なし発生せず

### 判定

成功

### 知見

GTK Windowライフタイム保持は主因ではない。

---

## WV-08-05 GTKイベントflush

### 目的

単発GTKイベント処理のみで問題が発生するか確認する。

### 実施内容

```rust
flush_gtk_events_bounded("WV-08-05");
```

### 結果

* GTK event flush start 出力確認
* GTK event flush completed 出力確認
* iterations=13
* デスクトップ左上に白いWindow表示
* Dock移動可能
* Dockサイズ変更可能
* マウス操作可能
* GTK Window上でマウス操作可能
* 応答なし発生せず

### 判定

成功

### 知見

以下は主因ではない可能性が高い。

* 単発GTKイベントflush
* gtk::main_iteration_do(false)
* GTK Window保持
* GTK Window表示

---

## WV-08-01 ～ WV-08-05総括

除外できた要因:

* gtk::init()
* gtk::Window::new()
* window.show_all()
* GTK Window保持
* 単発GTKイベントflush
* gtk::main_iteration_do(false)

有力候補:

* GTK Widget階層
* 継続GTKイベントポンプ
* WebKitGTK
* WebKitGTK統合処理
