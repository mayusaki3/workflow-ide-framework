## 検証結果概要

### WV-00

Dock埋め込み方式は条件付き成功。
Dock矩形取得、移動・リサイズ検知、Child Window追従が成立した。

### WV-01

WebView表示成功。
WebView生成、URL表示、eframeとの共存が成立した。

### WV-02

Windows環境での egui 共存成功。
Dock移動、リサイズ、タブ切替、Child Window追従が成立した。

### WV-03

Linux Wayland環境では不合格。
build_as_child() が利用できず、Linux向け方式選定が必要と判定した。

### WV-04

Linux標準方式選定完了。
build_gtk() + gtk::Fixed 方式を採用した。

### WV-05

条件付き成功。
イベントループ統合は主因ではなく、GTK Host Window構成が有力候補と判定した。

### WV-06

Waylandで応答なし再現、X11で非再現を確認。
Wayland系要因の可能性が高まった。

### WV-07

条件付き失敗。
Panel同期自体は成立したが、GTK Toplevel Window表示中に応答なしが発生した。

### WV-08

完了。
GTK、WebKitGTK、WebView、イベントポンプ、Visibility制御では応答なしを再現できなかった。

### WV-09

進行中。
WV-08で除外できなかった要素を切り分け中。

完了:

* WV-09-01 WV07_WV08差分分析
* WV-09-02 X11親子Window制御検証
* WV-09-03 GtkFixed階層検証
* WV-09-04 WebKitGTK + eframe / winit 共存検証
* WV-09-05 Native Child Window再導入検証
* WV-09-06-01 GTKイベントポンプ停止
* WV-09-06-02 GTKイベントポンプ最小化
* WV-09-06-03 Visibility同期GTKイベントポンプ停止
* WV-09-06-04 WebView set_visible継続実行停止

未実施:

* WV-09-06-05 Native Surface位置同期単独有効
* WV-09-06-06 Visibility同期単独有効
