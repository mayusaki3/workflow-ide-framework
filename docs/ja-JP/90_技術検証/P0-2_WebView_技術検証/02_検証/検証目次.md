# 検証目次

## 目的

P0-2 WebView 技術検証で実施する検証項目および進捗を管理する。

各検証の詳細な内容、実施結果、評価および設計判断は各 WV ドキュメントに記録する。

## 検証方針

本技術検証では、Workflow IDE Framework における Support Panel 実現方式として WebView を利用可能か確認する。

検証は以下の順序で実施する。

1. Windows における成立性確認
2. Linux における成立性確認
3. macOS における成立性確認
4. 複数 Native Surface 共存確認
5. IDE機能連携確認

## 検証環境

| 項目 | 内容 |
|---|---|
| 言語 | Rust |
| GUI Framework | egui |
| Window Backend | eframe |
| Dock System | egui_dock |
| WebView | wry |
| 対象OS | Windows / Linux / macOS |

## 検証結果概要

### WV-00

結果

- 条件付き成功

主な成果

- Dock矩形取得成立
- Child Window Overlay方式成立

### WV-01

結果

- 成功

主な成果

- WebView生成成功
- build_as_child方式成立

### WV-02

結果

- 条件付き成功

主な成果

- egui共存成立
- Floating禁止方式成立

### WV-03

結果

- 不成立

主な成果

- build_as_child() による WebView生成を試行
- Wayland環境で UnsupportedWindowHandle を確認
- wry 0.53.5 ソース調査を実施
- build_as_child() は X11専用であることを確認
- build_gtk() が Wayland/X11 共通方式であることを確認
- eframe から GTK Container を取得する方法は確認できなかった

### WV-04

結果

* 成功

主な成果

* build_gtk() による WebView生成成功
* Wayland環境で動作確認
* gtk::Fixed への配置成功
* move_() による移動成功
* set_size_request() によるリサイズ成功
* Linux標準方式を選定

備考

* 応答停止問題は配置方式ではなくイベントループ統合問題として切り出した

### WV-05

結果

* 条件付き成功

主な成果

* `build_gtk()` による WebView生成は成立する。
* WebView表示は成立する。
* eframe / winit EventLoop は継続動作する。
* GTK MainContext は継続駆動できる。
* 上限付き GTKイベント処理により、eframe と GTK の双方を停止させずに動作させられる。

備考

* 応答停止問題の主因は、イベントループ統合そのものではない。
* 主因候補は、GTK Host Window を独立トップレベルとして生成している構成である。

### WV-06

結果

* 成功

主な成果

* GTK Host Window の最小構成を確認
* build_gtk() が最小GTK構成でも動作することを確認
* GTK MainContext は継続動作することを確認
* GTKイベントポンプは継続動作することを確認
* Host Window種別変更では応答なし改善なし
* 入力イベント無効化では応答なし改善なし
* Wayland環境で応答なし再現
* X11環境で応答なし非再現

備考

* 応答なし問題は GTKイベントループ統合問題ではない
* Host Window構成問題でもない
* Wayland環境依存で発生する可能性が高い
* Hyper-V + Ubuntu + llvmpipe 環境で再現
* WV-07では GDK_BACKEND=x11 を前提として Panel同期方式を検証する
