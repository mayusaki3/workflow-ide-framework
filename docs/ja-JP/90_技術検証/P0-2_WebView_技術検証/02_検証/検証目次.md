# 検証目次

## 目的

P0-2 WebView 技術検証で実施する検証項目および進捗を管理する。

各検証の詳細な内容、実施結果、評価および設計判断は各 WV ドキュメントに記録する。

## 検証方針

本技術検証では、 Workflow IDE Framework における Support Panel 実現方式として WebView を利用可能か確認する。

検証は以下の順序で実施する。

1. Windows における成立性確認
2. Linux における成立性確認
3. Linux 応答なし原因特定
4. macOS における成立性確認
5. 複数 Native Surface 共存確認
6. IDE機能連携確認

## 検証結果概要

### WV-00

結果

* 条件付き成功

主な成果

* Dock矩形取得成立
* Child Window Overlay方式成立

### WV-01

結果

* 成功

主な成果

* WebView生成成功
* build_as_child方式成立

### WV-02

結果

* 条件付き成功

主な成果

* egui共存成立
* Floating禁止方式成立

### WV-03

結果

* 不成立

主な成果

* build_as_child() による WebView生成を試行
* Wayland環境で UnsupportedWindowHandle を確認
* wry 0.53.5 ソース調査を実施
* build_as_child() は X11専用であることを確認
* build_gtk() が Wayland/X11 共通方式であることを確認
* eframe から GTK Container を取得する方法は確認できなかった

### WV-04

結果

* 成功

主な成果

* build_gtk() による WebView生成成功
* Wayland環境で動作確認
* gtk::Fixed への配置成功
* move_() による移動成功
* set_size_request() によるリサイズ成功
* Linux標準方式を選定

### WV-05

結果

* 条件付き成功

主な成果

* build_gtk() による WebView生成成立
* WebView表示成立
* eframe / winit EventLoop継続動作
* GTK MainContext継続駆動
* 上限付きGTKイベント処理成立

### WV-06

結果

* 成功

主な成果

* GTK Host Window最小構成確認
* Wayland環境で応答なし再現
* X11環境で応答なし非再現

### WV-07

結果

* 条件付き失敗

主な成果

* Host Window位置・サイズ同期成立
* Host Window上マウス移動で応答なし再現

### WV-08

結果

* 完了

主な成果

* GTK完全無効化状態から段階的再導入を実施
* GTK / WebKitGTK / WebView単体要素では応答なしを再現しなかった
