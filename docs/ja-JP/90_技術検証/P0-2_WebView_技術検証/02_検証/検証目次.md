# 検証目次

## 目的

P0-2 WebView 技術検証で実施する検証項目および進捗を管理する。

各検証の詳細な内容、実施結果、評価および設計判断は各 WV ドキュメントに記録する。

## 検証方針

本技術検証では、 Workflow IDE Framework における Support Panel 実現方式として WebView を利用可能か確認する。

## WV-09進捗

WV-09配下の詳細検証は、[WV-09 Linux応答なし原因特定](WV-09_Linux応答なし原因特定.md) に集約する。

## 検証結果概要

### WV-00

Dock埋め込み方式は条件付き成功。
Dock矩形取得、移動・リサイズ検知、Child Window追従が成立した。

### WV-01

WebView表示成功。
WebView生成、URL表示、eframeとの共存が成立した。

### WV-02

Windows環境での egui 共存成功。
Dock移動、リサイズ、タブ切替、Child Window追従が成立した。

### WV-03

Linux Wayland環境では不合格。
build_as_child() が利用できず、Linux向け方式選定が必要と判定した。

### WV-04

Linux標準方式選定完了。
build_gtk() + gtk::Fixed 方式を採用した。

### WV-05

条件付き成功。
イベントループ統合は主因ではなく、GTK Host Window構成が有力候補と判定した。

### WV-06

Waylandで応答なし再現、X11で非再現を確認。
Wayland系要因の可能性が高まった。

### WV-07

条件付き失敗。
Panel同期自体は成立したが、GTK Toplevel Window表示中に応答なしが発生した。

### WV-08

完了。
GTK、WebKitGTK、WebView、イベントポンプ、Visibility制御では応答なしを再現できなかった。

### WV-09

完了。
WV-08で除外できなかった要素を切り分けたが、WV-09-06までの範囲では応答なしを再現できなかった。

同期処理単体では応答なしの主因を確認できなかったため、WV-10でWV-07再現時点との差分分析へ移行する。

### WV-10

未実施。
WV-07で応答なしが発生していた時点と現在の develop ブランチとの差分を分析する。
