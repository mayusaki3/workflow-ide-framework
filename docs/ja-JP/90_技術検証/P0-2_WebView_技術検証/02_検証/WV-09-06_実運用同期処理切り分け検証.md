# WV-09-06 実運用同期処理切り分け検証

## 目的

WV-09-05完了時点で残存している主因候補である実運用同期処理および Dock追従同期処理について切り分けを行い、Linux応答停止の発生条件を特定する。

## 背景

WV-09-05までの検証により、以下の要素は単独では応答停止を発生させないことを確認した。

* GTK Host Window
* Native Child Window
* GtkFixed階層
* WebKitGTK
* WebView配置制御
* WebView表示制御
* eframe / winit共存

そのため、残る主因候補は継続実行される同期処理群である。

## 検証方針

まず全ての同期処理を停止した構成を作成する。

その後、同期処理を1件ずつ再導入し、応答停止の再現有無を確認する。

## 検証構成

* GDK_BACKEND=x11
* eframe
* egui
* GTK Host Window
* Native Child Window
* WebKitGTK

## 検証項目

| 検証番号        | 項目                 | 期待結果     |
| ----------- | ------------------ | -------- |
| WV-09-06-01 | 全同期停止              | 応答停止有無確認 |
| WV-09-06-02 | Dock矩形取得処理のみ有効     | 応答停止有無確認 |
| WV-09-06-03 | Dock状態監視処理のみ有効     | 応答停止有無確認 |
| WV-09-06-04 | WebView位置同期処理のみ有効  | 応答停止有無確認 |
| WV-09-06-05 | Visibility同期処理のみ有効 | 応答停止有無確認 |
| WV-09-06-06 | タイマー同期処理のみ有効       | 応答停止有無確認 |
| WV-09-06-07 | イベント同期処理のみ有効       | 応答停止有無確認 |

## 判定基準

応答停止を再現した場合

* 当該同期処理を主因候補とする

応答停止を再現しない場合

* 次の同期処理へ進む
