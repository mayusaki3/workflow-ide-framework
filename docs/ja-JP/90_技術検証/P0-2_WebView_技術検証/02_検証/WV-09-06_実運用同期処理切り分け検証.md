# WV-09-06 実運用同期処理切り分け検証

## 目的

WV-09-05完了時点で残存している主因候補である実運用同期処理について切り分けを行い、Linux応答停止の発生条件を特定する。

## 背景

WV-09-05までの検証により、以下の要素は単独では応答停止を発生させないことを確認した。

- GTK Host Window
- Native Child Window
- GtkFixed階層
- WebKitGTK
- WebView配置制御
- WebView表示制御
- eframe / winit共存

ソースコード調査の結果、実運用時に継続実行される処理は主に以下であることを確認した。

- GTKイベントポンプ
- Native Surface位置同期
- Visibility同期

そのため、これらを主因候補として切り分けを実施する。

## 検証方針

実装上存在する同期処理を対象として切り分けを実施する。

まず対象同期処理を停止した状態を確認し、その後、単独有効状態で応答停止再現有無を確認する。

## 検証構成

- GDK_BACKEND=x11
- eframe
- egui
- GTK Host Window
- Native Child Window
- WebKitGTK

## 検証項目

| 検証番号        | 項目                 | 期待結果     |
| ----------- | ------------------ | -------- |
| WV-09-06-01 | GTKイベントポンプ停止 | 応答停止有無確認 |
| WV-09-06-02 | Native Surface位置同期停止 | 応答停止有無確認 |
| WV-09-06-03 | Visibility同期停止 | 応答停止有無確認 |
| WV-09-06-04 | GTKイベントポンプ単独有効 | 応答停止有無確認 |
| WV-09-06-05 | Native Surface位置同期単独有効 | 応答停止有無確認 |
| WV-09-06-06 | Visibility同期単独有効 | 応答停止有無確認 |

## 判定基準

応答停止を再現した場合

- 当該同期処理を主因候補とする

応答停止を再現しない場合

- 次の同期処理へ進む
