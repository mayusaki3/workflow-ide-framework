# Linux GUI Fallback 検証ケース

# 1. fallback script

## TC-P0-1D-001

### 目的

P0-1c を fallback 条件付きで起動し、Linux GUI backend 安定化が成立するか確認する。

### 前提

本検証は P0-1d 独自 Cargo project の検証ではない。

P0-1c の Cargo project を fallback 条件付きで実行する。

fallback script は P0-1c 配下で管理する。

### 手順

```bash
cd ~/workflow-ide-framework/docs/ja-JP/90_技術検証/P0-1c_Linux_Native_Window_Title_技術検証

chmod +x ./scripts/run_linux_fallback.sh

./scripts/run_linux_fallback.sh
```

### 期待結果

- P0-1c cargo run 成功
- GUI 起動
- libEGL/MESA/ZINK error 回避

# 2. renderer

## TC-P0-1D-002

### 目的

fallback 環境で egui renderer が正常動作するか確認する。

### 手順

1. P0-1c GUI 起動
2. Panel 表示確認
3. 日本語表示確認

### 期待結果

- Panel 表示
- 日本語表示

# 3. fallback 効果比較

## TC-P0-1D-003

### 目的

fallback 有無による差異を確認する。

### 手順

1. P0-1c を fallback なしで実行
2. P0-1c を fallback ありで実行
3. error 差異確認
4. GUI 差異確認

### 期待結果

- fallback により GUI backend 起動安定化
- fallback により libEGL/MESA/ZINK error 回避
- native title 日本語問題は残存

# 4. WebView 影響

## TC-P0-1D-004

### 目的

fallback が WebView に影響しないか確認する。

### 手順

1. P0-2 実行
2. WebView 表示確認
3. 日本語確認

### 期待結果

- WebView 表示
- WebKitGTK 起動
- fallback 環境で安定動作
