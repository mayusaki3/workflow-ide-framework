# 仕様分割方針メモ

本ファイルは、LLM が仕様分割方針を保持するための一時メモである。

正式仕様へ反映済みで不要になった内容は削除する。

---

# 現在の仕様作成方針

- 小さい単位で実装を進める
- 「仕様 → テスト仕様 → コード → テストコード → テスト」を細かく回す
- 実装前に巨大仕様を作り込まない
- まずアーキテクチャー寄り仕様を作成する
- 詳細仕様は必要タイミングで追加する

---

# 仕様構成方針

## 01_アーキテクチャー.md

- 全体構造を定義する
- Core 構造を固定化する
- Runtime / Workflow / GPU / Plugin / WebView / Event / State などの関係を定義する

## 02 以降

最初は「アーキテクチャー寄り」の仕様までを定義する。

詳細仕様は、実装直前で分割して追加する。

例:

- 02-1_Event_State_アーキテクチャー.md
- 02-2_Event_State_UI部品.md
- 02-3_Event_State_実行モデル.md

---

# 分割方針

## *_アーキテクチャー.md

- 方針固定用
- Core 構造を定義
- 実装依存を減らす
- 実装前提を固定しすぎない

## *_UI部品.md

- UI 詳細仕様
- Panel
- Viewport
- Layout
- Docking

## *_実行モデル.md

- Runtime 動作
- State 更新
- Event 発行
- Lifecycle
- Scheduler
