# 現在の方針

- 小さい単位で実装を進める
- 「仕様 → テスト仕様 → コード → テストコード → テスト」を細かく回す
- Runtime IDE として成立する構造を優先する
- Event / State・Command・Runtime 分離を重視する
- GPU Viewport は IDE コア構造として扱う
